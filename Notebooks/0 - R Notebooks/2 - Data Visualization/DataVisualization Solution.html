Q1.2

When aesthetics gets mapped to expressions, ggplot behaves as if a temporary variable was added to the data with with values equal to the result of the expression.

The result of salary > 100000 is a logical variable which takes values of TRUE or FALSE.

ggplot(df) +
  geom_point(aes(x = yrs.service, y = salary, color=salary > 100000))

Q1.3

For color, instead of using discrete colors, the continuous variable uses a scale that varies from a light to dark color.

For size, the sizes of the points vary continuously as a function of their size.

For shape, you get an error. (It doesnt make sense.)

ggplot(df) +
  geom_point(aes(x = yrs.service, y = salary, color=yrs.since.phd))


ggplot(df) +
  geom_point(aes(x = yrs.service, y = salary, size=yrs.since.phd))

Q2.1

The difference between the two is the stat function they use.

The default stat of geom_col() is stat_identity(). (You need both x and y, y representing the bar heights.)

The default stat of geom_bar() is stat_bin(). (You only need the x. y variable will be the result of the preprocessing done on x variable.)

Q2.3

As explained in Q1.2, since it is inside the aes() function, ggplot treats it as a new variable. (Categorical variable which only takes a single value)

This is the correct code:

ggplot(df) +
  geom_point(aes(x = yrs.service, y = salary), color = "green")
